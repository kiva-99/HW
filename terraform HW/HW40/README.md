# HW-40: Infrastructure as Code с Terraform

## Цель
Ознакомиться с основами инфраструктуры как код и практическим применением Terraform

## Что сделано
- ✅ Установлен и настроен Terraform v1.14.8
- ✅ Настроен провайдер Yandex Cloud (v0.196.0) через зеркало
- ✅ Создана конфигурация инфраструктуры:
  - Виртуальная сеть (VPC)
  - Подсеть в зоне ru-central1-a
  - Виртуальная машина на Ubuntu 24.04 LTS
- ✅ Оптимизирована стоимость ВМ:
  - Прерываемая ВМ (preemptible)
  - Гарантированная доля vCPU: 20%
  - RAM: 1 ГБ
  - Платформа: Intel Ice Lake (standard-v3)
- ✅ Итоговая стоимость: ~600 ₽/мес (экономия ~75%)

## Структура проекта
```
terraform HW/
├── main.tf              # Описание ресурсов
├── provider.tf          # Настройка провайдера
├── variables.tf         # Переменные
├── terraform.tfvars     # Значения переменных
├── key.json             # Ключ сервисного аккаунта (не коммитить!)
└── .gitignore
```

## Команды для развёртывания
```bash
terraform init           # Инициализация
terraform validate       # Проверка конфигурации
terraform plan           # Просмотр плана
terraform apply          # Применение
terraform destroy        # Удаление ресурсов
```
