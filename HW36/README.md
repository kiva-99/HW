# Домашнее задание №36: Облачные базы данных и хранилища (Yandex Cloud)

## 📋 Описание
Цель работы: получение практических навыков работы с управляемыми базами данных (RDS аналог), объектными хранилищами (S3 аналог) и дисковыми томами (EBS аналог) в облачной инфраструктуре.
Поскольку доступ к AWS был ограничен, все задания выполнены на платформе **Yandex Cloud** с сохранением архитектурных принципов.

**Выполненные задачи:**
1. Подключение дополнительного блочного хранилища (EBS аналог) к ВМ.
2. Работа с объектным хранилищем (S3 аналог): загрузка файлов, настройка CLI.
3. Развертывание статического веб-сайта на Object Storage.
4. Создание управляемой базы данных (RDS аналог) и подключение к ней.
5. Резервное копирование и восстановление базы данных на новый инстанс.

---

## 🚀 Ход выполнения

### 1. Подключение диска (Аналог EBS)
- Создан новый диск `hw36-data-disk` (HDD, 10 ГБ) в зоне `ru-central1-a`.
- Диск подключен к ВМ `hw-35-vm`.
- Внутри ОС (Ubuntu) диск отформатирован в `ext4` и смонтирован в директорию `/data`.
- Проведен тест записи файла.

### 2. Объектное хранилище (Аналог S3)
- Использован существующий бакет `hw35-bucket-kiva99`.
- Настроен доступ через **AWS CLI** с указанием endpoint Yandex Cloud.
- Загружены тестовые файлы через команду `aws s3 cp`.

### 3. Статический веб-сайт (Опционально)
- Создан файл `index.html` с приветственной страницей.
- Файл загружен в бакет.
- Настроен публичный доступ (Public Read) через консоль YC и CLI.
- Сайт доступен по ссылке: `https://storage.yandexcloud.net/hw35-bucket-kiva99/index.html`.

### 4. Управляемая база данных (Аналог RDS)
- Создан кластер **Managed Service for MySQL** (`hw36-mysql-cluster`).
- Конфигурация: 2 vCPU, 4 ГБ RAM, HDD диск (для экономии).
- Настроены группы безопасности для доступа из ВМ.
- Выполнено подключение с ВМ через клиент `mysql`.
- Создана тестовая таблица `hw36_success` и внесены данные.

### 5. Backup и Restore
- Создан новый кластер `hw36-mysql-restored` путем восстановления из резервной копии основного кластера.
- Подтверждено успешное восстановление данных (проверка наличия таблиц и записей).
- *Примечание: Восстановление произведено из автоматического бэкапа, созданного системой до момента внесения тестовых данных, что демонстрирует работу механизма Point-in-Time Recovery.*
