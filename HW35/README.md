# Домашнее задание №35: Облачные технологии (Yandex Cloud)

## 📋 Описание
Цель работы: знакомство с облачной инфраструктурой, получение практических навыков работы с консолью облачного провайдера. 
Поскольку доступ к AWS был ограничен, все задания были выполнены на платформе **Yandex Cloud** (аналог AWS).

**Выполненные задачи:**
1. Регистрация в облаке и активация пробного периода.
2. Развертывание виртуальной машины (аналог EC2).
3. Создание объектного хранилища (аналог S3).
4. Настройка масштабирования (Instance Groups).
5. Настройка бюджетов и оповещений о тратах.

---

## 🚀 Ход выполнения

### 1. Регистрация и настройка аккаунта
- Зарегистрирован аккаунт в Yandex Cloud.
- Активирован грант на 4000₽ для новых пользователей.
- Настроен платежный профиль.

### 2. Виртуальная машина (Аналог Amazon EC2)
- Создана ВМ `hw-35-vm`.
- **Параметры:**
  - ОС: Ubuntu 22.04 LTS
  - Платформа: Intel Ice Lake
  - vCPU: 2, RAM: 1 ГБ
  - Диск: 10 ГБ SSD
  - Зона доступности: `ru-central1-a`
- Настроены правила безопасности (SSH доступ).

### 3. Объектное хранилище (Аналог Amazon S3)
- Создан бакет для хранения файлов.
- Настроены права доступа.
- *Примечание: Файлы загружены через консоль.*

### 4. Масштабирование (Аналог Auto Scaling Groups)
- Создана **Группа виртуальных машин** (`hw35-autoscale-group`).
- **Конфигурация для минимизации затрат:**
  - Тип масштабирования: Фиксированный (размер 1) / Готовность к автоскейлингу.
  - Шаблон ВМ: Прерываемая ВМ (Spot instance) для экономии.
  - Диск: HDD (для удешевления).
  - Публичный IP: Не назначен (доступ через внутреннюю сеть/бастион).
- Группа успешно создана и находится в статусе `Active`.

### 5. Бюджеты и оповещения
- Создан бюджет в Billing Center.
- Лимит: 300₽ (~$3.5).
- Порог оповещения: 50% (150₽).
- Статус: `Active`.
